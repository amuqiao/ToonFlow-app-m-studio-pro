## 结论

AI 漫剧创作流程通常可以压缩为一条主线：

**原著导入 → 事件提取 → 故事骨架 → 改编策略 → 剧本 → 资产提取/塑角造景 → 导演规划 → 分镜表/分镜图 → 视频提示词 → 视频生成 → 拼接导出**

Toonflow 基本遵循这条链路，只是将其拆成：

- `ScriptAgent` 负责剧本开发
- `ProductionAgent` 负责生产流程
- 工作台负责组织分镜、素材、视频轨与生成结果

## 一、通用 AI 漫剧创作流程

### 1. 项目策划

先确定项目边界，包括：

- 题材与世界观
- 平台规格
- 集数与单集时长
- 画幅与视觉风格
- 付费策略与目标受众

这一阶段的目标，是把后续创作的约束条件先固定下来。

### 2. 原著整理与事件提取

将小说、大纲或剧情设定导入系统，拆解为结构化事件数据，例如：

- 章节范围
- 主要冲突
- 人物关系
- 情绪变化
- 伏笔与反转节点

这一阶段的目标，是为后续剧本改编提供可检索、可调用的上下文。

### 3. 剧本开发

剧本开发通常分三层：

- 故事骨架：确定整体三幕或分集结构
- 改编策略：确定删改、强化、节奏与平台适配原则
- 结构化剧本：输出分集内容、场次、对白与动作

这一阶段的目标，是把原始故事改写为可生产的短剧文本。

### 4. 资产设计

从剧本中提取可视化资产，并生成图像制作所需描述，包括：

- 角色
- 场景
- 道具
- 衍生资产

这一阶段的目标，是为分镜图和视频生成准备统一的视觉参考。

### 5. 导演规划

在进入分镜前，先确定整体导演语言，例如：

- 镜头风格
- 运镜节奏
- 光影氛围
- 情绪曲线
- 风格一致性原则

这一阶段的目标，是让后续分镜和视频提示词保持统一审美。

### 6. 分镜生产

将剧本拆成镜头级表达，通常包含：

- 分镜表
- 分镜面板
- 分镜图
- 分镜与资产关联

这一阶段的目标，是把剧本文本转成可执行的镜头计划。

### 7. 视频制作

基于分镜图、角色/场景参考图、视频提示词和模型配置，生成：

- 单条视频片段
- 视频轨
- 首尾帧参考
- 多参考视频任务

这一阶段的目标，是把静态分镜转化为动态镜头。

### 8. 拼接与出片

最后汇总：

- 视频片段
- 配音
- 音效
- 字幕
- 封面与导出结果

这一阶段的目标，是得到可交付的成片。

## 二、Toonflow 中的实际对应关系

### 1. 总体主线

Toonflow README 对主流程的概括是：

**策划 → 编剧 → 分镜 → 出片**

并在快速上手中给出了更细的执行顺序：

1. 新建项目并导入原著
2. 执行章节事件提取
3. 进入 `ScriptAgent` 生成故事骨架、改编策略与结构化剧本
4. 切换到 `ProductionAgent` 组织分镜、素材与视频节点
5. 对分镜图精调后回流工作台，完成视频拼接与导出

## 四、从工程实现角度看 Toonflow 的流程特点

### 1. 不是线性工具，而是工作台式流程

Toonflow 虽然有主流程，但不是严格的一步一页向前推进。它更像一个围绕生产对象组织的工作台，可以在以下对象之间来回切换：

- 剧本
- 资产
- 分镜
- 视频节点

这也是 README 中“类无限画布生产工作台”的含义。

### 2. Agent 与工作台分工明确

在 Toonflow 里：

- Agent 更偏“内容生成与决策”
- 工作台更偏“结果组织与人工调整”

也就是说，Agent 负责产出，工作台负责编排、回看、校正和继续生产。

### 3. 分镜是关键中间层

从系统设计上看，分镜是连接剧本与视频的核心中间产物：

- 向上承接剧本
- 向下驱动视频提示词与视频生成
- 横向关联角色、场景、道具等资产

因此分镜质量会直接影响后续视频生成稳定性和成片一致性。

## 五、理解本项目时最重要的一条主线

如果只保留一条最核心的理解路径，可以记成：

**原著内容先被改写成结构化剧本，再被拆成资产与分镜，最后由工作台把分镜、参考素材和视频轨组织成成片。**

对 Toonflow 来说，最关键的不是单个模型调用，而是：

- 剧本是否结构化
- 资产是否可复用
- 分镜是否可执行
- 视频轨是否可编排

这四层是否连起来，决定了整条 AI 漫剧生产链路是否稳定。
