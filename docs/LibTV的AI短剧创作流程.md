# LibTV 的 AI 短剧创作流程

> **文档职责**  
> 梳理 LibTV 面向 AI 短剧 / 漫剧创作的典型工作流程，并将其拆解为清晰的阶段、输入输出和工作台动作，便于对标、学习和后续产品分析。  
>
> **适用场景**  
> 适用于理解 LibTV 的创作链路、与 Toonflow 做流程对比、为产品设计提供参考，以及为团队成员建立对 AI 短剧生产平台的全局认知。  
>
> **阅读目标**  
> 帮助读者快速理解 LibTV 如何把“剧本 → 分镜 → 图片 → 视频 → 出片”组织在同一套工作台中，并明确每个阶段的核心目标与典型动作。  
>
> **目标读者**  
> 适用于产品经理、后端开发、前端开发、Agent 工作流设计者，以及需要研究 AI 视频创作平台流程的协作者。  
>
> **维护规范**  
> 本文优先记录稳定的主流程与平台能力，不记录一次性活动页面或短期营销话术。当前版本由于指定飞书指南无法公开访问，内容基于公开可访问的 LibTV 产品介绍、发布资料与第三方功能总结整理；若后续获得官方指南访问权限，应优先按官方文档校准流程细节、术语与顺序。

## 结论

LibTV 的 AI 短剧创作流程可以压缩为一条主线：

**项目创建 → 剧本输入 / 剧情设定 → 分镜脚本生成 → 分镜图批量生成 → 视频片段批量生成 → 片段筛选与回修 → 拼接导出**

它的核心特点不是单点模型调用，而是把以下几类能力放进同一套无限画布工作台中：

- 剧本与脚本节点
- 图片与视频节点
- 分镜批量生产能力
- 专业图像 / 视频工具
- Agent 可调用的 Skill 能力

## 一、LibTV 的流程定位

从公开资料看，LibTV 不是单一的“生视频工具”，而是一套面向短剧 / 漫剧创作的整合式视频工作台。它将剧本、分镜、图片、视频、音频等对象放到同一张无限画布上，通过节点式工作流组织创作过程。

对 AI 短剧来说，这意味着：

- 剧本不是平台外部准备好再导入的孤立文本
- 分镜不是单张单张手工生成的离散结果
- 视频也不是只做最终一步的单次推理

而是把从文本到成片的多个中间步骤都纳入同一生产界面。

## 二、LibTV 的典型 AI 短剧创作流程

### 1. 项目创建

首先新建项目，进入无限画布工作台。

这一阶段的目标是：

- 确定项目主题
- 确定创作类型
- 准备后续剧本、分镜、图片、视频节点的组织空间

在 LibTV 的语境里，画布既是流程容器，也是结果容器。

### 2. 剧情设定 / 剧本输入

进入画布后，先提供创作起点。公开介绍显示，LibTV 支持以下几类输入方式：

- 直接输入剧情描述
- 上传剧本
- 上传角色参考图
- 上传参考视频

这一阶段的目标，是把故事意图转化为后续可批量处理的脚本或分镜基础。

### 3. 分镜脚本生成

在 LibTV 的公开介绍里，“剧本转分镜脚本”是整条链路的关键中间层。也就是说，平台并不是直接从剧本一步跳到最终视频，而是先形成可以驱动后续生图和生视频的分镜脚本。

这一阶段通常承接：

- 剧情描述
- 剧本内容
- 角色参考
- 基础镜头意图

并输出：

- 分镜脚本
- 镜头拆分结果
- 后续可批量执行的分镜单元

### 4. 分镜图批量生成

在形成分镜脚本后，LibTV 会将脚本进一步转成分镜图。公开资料强调它支持：

- 一键生成 9 / 25 宫格分镜
- 批量生图
- 每条分镜单独调整后重新生成

这一阶段的目标，是把镜头级文本描述转成视觉参考，并为视频生成提供首帧或画面基准。

### 5. 视频片段批量生成

在分镜图基础上，LibTV 会继续向下执行视频生成。公开资料中明确提到：

- 分镜图可继续批量转视频片段
- 支持多种视频模型切换
- 各环节支持单条回修和重新生成

这一阶段的目标，是围绕每条分镜得到可用的视频片段，而不是一次性直接得到完整成片。

### 6. 片段筛选与回修

LibTV 的流程不是“生成完就结束”，而是允许在中间层不断回修。这类回修动作通常包括：

- 对单条分镜脚本微调
- 对单张分镜图重生
- 对单段视频片段重新生成
- 切换模型或工具后再次执行

这一阶段的目标，是在进入最终导出前，把片段质量稳定下来。

### 7. 视频片段汇总与导出

公开资料显示，LibTV 已经覆盖“从剧本到视频片段”的主流程，并提供基础视频剪辑能力，但最终成片流程仍强调“汇总片段后导出使用”。

因此这一步更适合理解为：

- 收敛视频片段
- 组织顺序与节奏
- 必要时补充音频、字幕、剪辑
- 输出成片

## 三、LibTV 的流程特点

### 1. 以画布为中心，而不是以单个工具为中心

LibTV 的核心不是某一个“文生视频模型”，而是围绕无限画布组织整个项目。它支持的基础节点类型至少包括：

- 文本
- 图片
- 视频
- 音频
- 脚本

这意味着它更像一个视频创作工作台，而不是单用途生成器。

### 2. 中间层非常重：分镜脚本和分镜图都不是可有可无

LibTV 的公开产品描述反复强调：

- 剧本转分镜脚本
- 分镜脚本转分镜图
- 分镜图转视频

这说明它的设计重点，是通过中间层提高镜头一致性、连贯性和批量生产效率。

### 3. 允许批量生产，也允许单条回修

对短剧创作而言，真正可用的平台不是只能批量跑，而是：

- 可以批量推进
- 也允许单条局部修复

LibTV 从公开资料看就是按这个思路组织的，这对漫剧 / 短剧这种多镜头、多片段内容尤其重要。

### 4. 兼容人类创作者和 Agent 两种入口

LibTV 的另一个明显特征，是同时强调：

- 人类创作者在画布中手动创作
- Agent 通过 Skill 自动调用能力

这意味着它既是创作工作台，也是面向自动化工作流的执行平台。

## 四、阶段输入 / 输出表

这张表用于说明每个阶段拿什么进来、产出什么出去。

| 阶段 | 主要输入 | 核心处理 | 主要输出 |
| --- | --- | --- | --- |
| 项目创建 | 创作主题、项目目标 | 建立画布与项目容器 | 项目工作区 |
| 剧情设定 / 剧本输入 | 剧情描述、剧本、角色图、参考视频 | 确定创作起点与叙事基础 | 剧本素材、脚本输入 |
| 分镜脚本生成 | 剧情描述、剧本、角色参考 | 拆分镜头、形成分镜脚本 | 分镜脚本 |
| 分镜图批量生成 | 分镜脚本、角色/风格参考 | 将镜头文本转成视觉参考 | 分镜图 |
| 视频片段批量生成 | 分镜图、视频模型、视频提示信息 | 为每条分镜生成动态片段 | 视频片段 |
| 片段筛选与回修 | 分镜脚本、分镜图、视频片段 | 单条回修、重生、模型切换 | 可用片段集合 |
| 视频片段汇总与导出 | 视频片段、必要剪辑与补充素材 | 组织顺序、导出结果 | 成片或可交付片段 |

## 五、如果把 LibTV 的流程压缩成产品语言

如果只保留一句最核心的描述，可以写成：

**LibTV 通过“无限画布 + 节点工作流 + 批量分镜生产”，把 AI 短剧创作从单次模型调用，变成了可组织、可回修、可自动化的项目式生产流程。**

## 六、对 Toonflow 的参考价值

如果站在对比分析角度，LibTV 对 Toonflow 有几个很直接的参考点：

- 更强调无限画布作为创作中心
- 更强调分镜脚本到分镜图再到视频的中间层闭环
- 更强调批量执行与单条回修并存
- 更强调人类创作者与 Agent 双入口

因此，研究 LibTV 的价值，不在于模仿单个功能，而在于理解它如何把“生成能力”组织成“生产流程”。
